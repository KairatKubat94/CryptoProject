📝 Шифр Цезаря

Описание: Каждая буква смещается на фиксированное количество позиций в алфавите.

Ключ: Смещение (обычно 3)

Стойкость: Очень низкая - всего 26 вариантов

Пример: A → D, B → E, C → F (при смещении 3)

Результат:

-

Пошаговое преобразование:

Введите текст и нажмите "Шифровать"

🔒 AES (Advanced Encryption Standard)

Описание: Симметричный алгоритм - один ключ для шифрования и расшифрования.

Размер ключа: 256 бит

Размер блока: 128 бит

Стойкость: Очень высокая - стандарт для защиты данных

Использование: Защита файлов, конфиденциальная переписка

Результат шифрования:

-

Процесс шифрования:

Введите текст и пароль, затем нажмите "Шифровать"

🔑 RSA (Rivest-Shamir-Adleman)

Описание: Асимметричный алгоритм - разные ключи для шифрования и расшифрования.

Открытый ключ: Используется для шифрования и проверки подписей

Секретный ключ: Используется для расшифрования и подписи документов

Размер ключа: Минимум 2048 бит (рекомендуется 4096)

Использование: Цифровые подписи, обмен ключами, HTTPS

Результат шифрования:

-

Процесс:

Ключи не сгенерированы

💡 Практические советы

🚫 Caesar - Не используйте!

Исторический интерес только. Легко подбирается перебором всех 26 вариантов за секунду.

✅ AES - Используйте для защиты данных

Стандарт NIST. Используйте для шифрования файлов, базы данных, конфиденциальной переписки.

🔐 RSA - Используйте для подписей

Используйте для цифровых подписей, обмена ключами AES, HTTPS сертификатов.

🔑 Управление ключами

Никогда не передавайте секретные ключи. Используйте надежные пароли для AES. Храните ключи безопасно.

🛡️ Best Practices

Комбинируйте RSA для обмена ключами + AES для шифрования данных. Используйте минимум 2048 бит для RSA.

🔍 Хеширование

Для паролей используйте bcrypt/Argon2, а не AES. Для проверки целостности - SHA256.
