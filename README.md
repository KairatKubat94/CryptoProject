## 🎯 Описание

Визуализатор демонстрирует работу трёх классических алгоритмов шифрования:

### 📝 **Caesar Cipher (Шифр Цезаря)**

- **Тип:** Шифр подстановки
- **Ключ:** Смещение (1-25)
- **Описание:** Каждая буква смещается на фиксированное количество позиций
- **Стойкость:** Очень низкая (всего 26 вариантов)
- **Использование:** Только историческое/учебное

### 🔒 **AES (Advanced Encryption Standard)**

- **Тип:** Симметричный шифр
- **Ключ:** Пароль (преобразуется в 256-битный ключ)
- **Размер блока:** 128 бит
- **Описание:** Текст разбивается на блоки и каждый блок шифруется отдельно
- **Стойкость:** Очень высокая - стандарт NIST
- **Использование:** Защита данных, конфиденциальная переписка

### 🔑 **RSA (Rivest-Shamir-Adleman)**

- **Тип:** Асимметричный шифр
- **Ключи:** Пара ключей (открытый + секретный)
- **Размер:** 2048 или 4096 бит
- **Описание:** Разные ключи для шифрования и расшифрования
- **Стойкость:** Очень высокая - сложность основана на факторизации
- **Использование:** Цифровые подписи, обмен ключами, HTTPS

## 🎓 Алгоритмы

### 1️⃣ Caesar Cipher

**Принцип работы:**

```
Исходный текст: "A B C"
Смещение: 3
Результат:      "D E F"

A (код 65) → смещение на 3 → D (код 68)
```

**Шаги в приложении:**

1. Выбор смещения (1-25)
2. Для каждой буквы применяется смещение
3. Остальные символы остаются без изменений

### 2️⃣ AES-256

**Принцип работы:**

```
1. Пароль преобразуется в 256-битный ключ
2. Генерируется случайный IV (вектор инициализации)
3. Текст разбивается на блоки по 128 бит
4. Каждый блок шифруется отдельно
5. Результат кодируется в Base64
```

**Шаги в приложении:**

1. Ввод пароля (преобразуется в ключ)
2. Подготовка текста
3. Инициализация шифра
4. Шифрование блоков
5. Кодирование результата

### 3️⃣ RSA-2048/4096

**Принцип работы:**

```
1. Генерируются два больших простых числа p и q
2. n = p × q
3. φ(n) = (p-1) × (q-1)
4. e = 65537 (открытый показатель)
5. d рассчитывается: e × d ≡ 1 (mod φ(n))

Открытый ключ: (e, n)
Секретный ключ: (d, n)

Шифрование: C = M^e mod n
Расшифрование: M = C^d mod n
```

**Шаги в приложении:**

1. Выбор размера ключа (1024, 2048, 4096 бит)
2. Генерирование пары ключей
3. Вход текста для шифрования
4. Шифрование открытым ключом
5. Вывод зашифрованных данных

## 💡 Практические советы

### 🚫 Caesar - Не используйте!

- Исторический интерес только
- Легко подбирается перебором за секунду
- Используется только для обучения

### ✅ AES - Используйте для защиты данных

- Стандарт NIST для шифрования
- Используйте для защиты файлов, БД, переписки
- Используйте надежные пароли (минимум 12 символов)

### 🔐 RSA - Используйте для подписей

- Используйте для цифровых подписей
- Используйте для обмена ключами AES
- Минимум 2048 бит, лучше 4096

### 🔑 Управление ключами

- **Никогда** не передавайте секретные ключи
- Храните ключи в безопасном месте
- Используйте надежные пароли
- Регулярно меняйте пароли

### 🛡️ Best Practices

- Комбинируйте RSA + AES (обмен ключами AES по RSA)
- Используйте bcrypt/Argon2 для паролей, не AES
- Используйте SHA256 для хешей и подписей
- Регулярно обновляйте криптографические библиотеки

## ❓ Часто задаваемые вопросы

**Q: Почему Caesar так нестоек?**
A: Потому что существует только 26 вариантов смещения. Даже компьютер 1980 года может перебрать их все за миллисекунды.

**Q: Может ли AES быть взломан?**
A: Теоретически - есть уязвимости, но практически нет. Перебор всех ключей займет больше времени, чем существует Вселенная.

**Q: Зачем нужен RSA, если есть AES?**
A: AES требует обмена ключами. RSA используется, чтобы безопасно передать ключ AES.

**Q: Почему RSA медленный?**
A: Потому что требует операций над очень большими числами (2048+ бит).
